# NEXORA Blueprint v1.0

## 0. North Star

NEXORAは、AI・仕事効率化・便利ツールをテーマに、SNS・ブログ・アフィリエイト・広告収益で月100万円を目指す個人事業である。

NEXORA AI OSは、NEXORAの運営を最小労力で回すための社内司令塔である。

最重要原則：

> AIが考え、AIが作り、AIが分析し、人は承認だけする。

## 5. NEXORA AI OSの役割

NEXORA AI OSは、販売用SaaSとして先に作るものではない。

まずは、健さん本人が毎日使い、SNS・ブログ・アフィリエイト運営を効率化し、収益を作るための内部ツールとして育てる。

その結果、2〜3年後、またはそれより早く、外部販売できる品質まで成長させる。

## 7. 基本ワークフロー

1. AIが世界中の情報を収集
2. AIが市場・SNS・検索トレンドを分析
3. AIが伸びそうなテーマを提案
4. AIがASP案件を選定
5. AIがASP内で使う検索ワードを提案
6. AIがASP検索URLを提示
7. AIがSNS・ブログ・動画用コンテンツを生成
8. AIが画像・動画構成を生成
9. AIが投稿時間を分析
10. 健さんが確認・承認
11. 承認済みのみ自動投稿または予約投稿
12. AIが結果を分析
13. AIが改善案を出す
14. Memory AIが学習する

---

## 8. AIの役割

### Think AI
市場・SNS・検索・ニュース・海外情報を分析し、今日伸びるテーマを提案する。

### Revenue AI
ASP案件、報酬、カテゴリ、導線、収益期待値を分析し、どの案件を使うべきか提案する。

### Creator AI
Instagram、Threads、X、TikTok、YouTube Shorts、ブログ、Canva構成、CapCut台本を作成する。

### Memory AI
過去投稿、クリック率、保存率、収益、案件相性、成功パターンを記憶する。

### Approval AI
投稿前に誤字、誇張、規約リスク、ブランド毀損、案件導線の自然さを確認する。

## 9. 主要画面

### Home Dashboard
毎朝最初に開く画面。

表示内容：
- 今月売上
- 今月目標
- 達成率
- 今日の投稿予定
- 承認待ち件数
- おすすめテーマ
- おすすめASP案件
- 今日の収益期待値

### Revenue Brief
AIが毎朝、収益期待値が高いテーマと案件を提案する画面。

表示内容：
- 今日のおすすめテーマTOP3
- 収益期待値
- 伸びる理由
- 競合状況
- おすすめASP案件
- 投稿すべき媒体
- 想定導線

### Affiliate Hub
提携済み・申請中・未提携案件を管理する画面。

管理項目：
- 案件名
- ASP名
- カテゴリ
- 報酬
- EPC
- CVR
- 承認率
- 提携状況
- 案件URL
- SNS向き
- ブログ向き
- Revenue Score
- メモ

### ASP Search Intelligence
テーマに応じて、ASP内で検索すべきワードと検索リンクを提案する画面。

### Trend Engine
世界・日本のトレンドを収集し、今伸びるテーマを提案する画面。

対象：
- Google Trends
- X
- Threads
- Instagram
- TikTok
- YouTube
- Reddit
- Product Hunt
- AIニュース
- GitHub Trending

### Content Studio
1テーマから複数媒体用コンテンツを自動生成する画面。

生成対象：
- Instagramカルーセル
- Instagramキャプション
- Threads投稿
- X投稿
- TikTok台本
- YouTube Shorts台本
- ブログ記事
- Canva指示
- CapCut指示
- ハッシュタグ
- CTA

### Approval Center
AIが作成した投稿を確認・承認する画面。

ステータス：
- 承認待ち
- 承認済み
- 修正依頼
- 保留
- 却下
- 投稿予約済み
- 投稿済み

健さんが行うこと：
- 承認
- 修正
- 保留
- 却下
- 投稿許可

### Publishing Scheduler
承認済み投稿を、AIが分析した最適時間に投稿または予約する画面。

### Analytics
投稿結果と収益を確認する画面。

表示内容：
- 投稿数
- 表示回数
- クリック数
- CTR
- CV
- 売上
- ASP別成果
- 媒体別成果
- 案件別成果

### Memory
過去の成功・失敗パターンを記憶する画面。

### Settings
外部連携・API・アカウント管理画面。

## 10. Phase別ロードマップ

### Phase 1：今月｜3万円を作る
目的：投稿作成と案件選定をAI化し、健さんは確認して投稿するだけにする。

実装：
- Home Dashboard
- Revenue Brief
- Affiliate Hub
- ASP Search Intelligence
- Content Studio基礎版
- Approval Center基礎版

運用：
- AIが投稿案を作る
- 健さんが確認して投稿
- ASP案件を手動登録
- 投稿結果を手動記録

### Phase 2：2〜3か月目｜10万〜30万円
目的：トレンド分析と案件選定を強化する。

実装：
- Trend Engine
- Revenue Score
- 投稿カレンダー
- Analytics基礎版
- ASP案件マスター拡張

### Phase 3：4〜6か月目｜60万〜100万円
目的：承認後の投稿予約・分析・改善を半自動化する。

実装：
- Publishing Scheduler
- Google Sheets連携
- WordPress連携
- SNS予約投稿
- Memory AI
- Revenue Prediction

### Phase 4：1〜2年
目的：外部販売を見据えたSaaS基盤を作る。

実装：
- ユーザー認証
- チーム機能
- Stripe決済
- プラン管理
- 顧客別データ管理
- SaaS用UI

## 13. GitHub管理方針

### Issue運用
1 Issue = 1機能。

Issueには必ず以下を記載する。

- Why
- Goal
- Revenue Impact
- Implementation
- Done Criteria

### Project運用
Kanbanで管理する。

列：
- Todo
- In Progress
- Review
- Done

### Release運用
主要アップデートごとにバージョンを付ける。

例：
- v2.1：Affiliate Hub
- v2.2：ASP Search Intelligence
- v2.3：Content Studio
- v2.4：Approval Center
